## 1. 核心理念

**错误是进化素材，不是失败。**

- 认识错误 → 记住错误 → 改正错误，是加速进化的最快路径
- 不追求更多信息或功能，追求**减少重复犯错**
- 任务处理原则：**完整落地，不做最小闭环**

---

## 2. 进化循环（4层架构）

```
输入层 → 孵化层 → 固化层 → 输出层
  ↓        ↓        ↓        ↓
接收任务  执行+记录  错误修复   能力输出
         +自检     技能固化   +进化
                   知识入库
```

### 输入层
- 用户任务（飞书 DM）
- Cron 定时任务（每日选股、Clawvard 考试等）
- Bot-to-Bot 协作（刘大虾）

### 孵化层
- `ralph_iteration.py` — 自主开发循环（PRD → ACTOR → Review）
- `dojo.py` — 复盘与知识沉淀
- `distill_learnings()` — 核心沉淀函数

### 固化层
- **错误修复**：`log-error-correction` Cron（每12小时）
- **技能固化**：`skill-cycle-optimizer`（每2小时）
- **每日蒸馏**：`daily-distill`（每日）

### 输出层
- 选股系统推送（飞书）
- 自我进化报告（飞书）
- Obsidian vault 写入（ learnings / daily / system / projects）

## 3. 记忆系统架构

### 分层记忆（由近到远）

| 层级 | 工具 | 用途 |
|------|------|------|
| 工作记忆 | LCM（会话压缩） | 当前任务上下文 |
| 短期记忆 | Hermes fact_store | 当前会话内的关键事实 |
| 中期记忆 | mem9 语义记忆 | 跨会话重要事实 |
| 长期记忆 | Skills + GitHub | 可执行知识与技能 |

### 记忆写入规则

**触发写入（主动）：**
- 用户明确说"记住这个"
- 发现了环境事实（OS、工具、API）
- 发现了用户偏好或习惯
- 解决了非平凡问题

**触发写入（自动）：**
- 每次复杂任务完成后 → `distill_learnings()`
- 每次错误后 → `log-error-correction`
- 每日蒸馏 → `daily-distill`

### 记忆宫殿（Dojo）

6个房间 × 37个桩位，用场景记忆法管理高频知识。

- `enter <room>` — 进入房间
- `walk <room> <n>` — 走到第n个桩位
- `tri <trigger>` — 触发记忆

## 4. 技能系统（Skills）

### 技能注册表

Skills 在 `~/.hermes/skills/` 目录下，触发时自动路由。

**关键技能：**
- `self-evolution-system` — 错误追踪+知识输入+技能固化
- `skill-cycle-optimizer` — 每2小时测试一个技能
- `log-error-correction` — 每12小时分析未关闭错误
- `daily-distill` — 每日会话蒸馏
- `brainstorming` — 行动前必用的规划技能

### 技能自动路由

Gateway 的 `skill_router.py` 在消息入口自动匹配相关技能。
触发条件写在每个技能的 SKILL.md frontmatter 里。

## 5. 核心错误处理机制

### 错误分类

| 类型 | 定义 | 处理方式 |
|------|------|----------|
| Retrieval | 文档/记忆中没有就不写 | 承认不知道，不脑补 |
| Reasoning | 不发明选项，不脑补需求 | 严格按给定的逻辑走 |
| Execution | 数据结构/API 调用错误 | 记录具体错误，修复代码 |

### 错误复盘模板

```markdown
## 错误记录
- 日期：
- 错误描述：
- 根因：
- 修复方案：
- 固化措施（技能/记忆）：
- 避免重复：
```

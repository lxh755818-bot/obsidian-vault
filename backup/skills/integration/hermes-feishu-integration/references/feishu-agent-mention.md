## 1. 核心结论（速查）

一句话执行准则：飞书群内 @Agent 必须用 `<at user_id="">显示名</at>` XML 标签；被 @回复时需携带发起方 @标签发回原群，形成**对话闭环**；仅群聊生效，单聊不适用。

| 参数 | 强制要求 | 校验规则 |
| ------- | ---- | ------------------------------ |
| `<at>` 标签 | 是 | 完整闭合，无嵌套，仅保留 user_id 属性（可为空） |
| user_id | ✅ 必须填写 | 必须以 `ou_` 开头，32位十六进制字符串。**id留空只显示蓝色但不推送通知**，必须填真实open_id才能触发通知 |
| 群 ID | 是 | 必须以 `oc_` 开头，对应普通群 / 部门群 |
| **@前后必须有文字** | ✅ 关键 | `<at>` 标签前后必须有文字包裹，否则飞书可能不投递通知 |
| 群聊直接发消息 | 是 | 不需要私聊，群里发消息即可 |

**✅ 正确格式**
```
<at user_id="ou_5de19270f8825dae4fd13f9f19d229ef">刘二虾</at>
```

**❌ 错误格式**
- 普通 `@名字` → 不触发
- `<at user_id="">名字</at>` → 显示蓝色但**不推送通知**
- @前后无文字：`@刘二虾`（裸@失效）

## 2. ⚠️ openclaw.json 群配置要求

```json
"accounts": {
  "liu2": {
    "groupPolicy": "allow",
    "groupAllowFrom": ["*"],
    "groups": { "*": { "enabled": true } }
  }
}
```

| 配置项 | 必需 | 说明 |
| -------------- | --- | ------------------- |
| groupPolicy | ✅ | 设为 `"allow"` |
| groupAllowFrom | ✅ | 设为 `["*"]`；空数组 = 全部拒绝 |
| groups["*"] | ✅ | `"*"` 表示允许所有群 |

**常见坑：**
- `groupPolicy: "deny"` → 完全忽略群消息
- `groupAllowFrom: []` → 所有消息被拒绝
- 日志显示 `"sender xxx not allowed in group"` = groupAllowFrom 问题

## 4. 全链路操作步骤

### 发起方
1. 从通讯录获取已验证的 `open_id`
2. 使用 `<at user_id="ou_xxx">名字</at>` 格式发送消息

### 被@后回复
1. 提取发起方 `open_id` 和原群 ID
2. 构造回复：`<at user_id="ou_xxx">发起方名字</at> {回复内容}`
3. 发回原群 → 完成闭环

## 7. Bot-to-Bot Relay 原理

飞书限制：Bot 之间消息不触发事件。

OpenClaw 解法：Hook 拦截 → 解析 `<at>` 标签 → 构造合成事件 → sessions_send 转发 → 回传结果

## 8. 错误排查

### 检查顺序（优先级从高到低）

1. **openclaw.json**：groupPolicy / groupAllowFrom / groups 三项是否齐全
2. **格式**：是否用 `<at user_id="">`
3. **ID**：user_id 是否以 `ou_` 开头（或留空）
4. **群类型**：必须是群聊
5. **工具**：`channel: feishu` + `target: chat:群ID`

### 快速自检

- @不响应？→ 查 groupAllowFrom
- 日志显示 `"sender not allowed"`？→ groupAllowFrom 问题

---

## 9. 高频坑点

- 必须用 `message` 工具（机器人身份），不能用 `feishu_im_user_message`（用户身份）
- 并发限制：每次 @ 控制在 2 个以内，间隔几秒再发
- 回复闭环：被@后不带@的回复会导致对话链断裂
- **记忆口诀**：回复群友，必带@

## 10. 飞书群成员 API 要点

**重要**：`GET /im/v1/chats/{chat_id}/members` 只返回真实用户，不返回 bot。

正确查询方式：
```bash
# 查询群信息（含 bot 数量）
GET /im/v1/chats/{chat_id}
# 响应中 bot_count 字段表示群里机器人数量
# user_count 字段表示真实用户数量
```

群里实际有 5 个机器人，但成员列表 API 只返回刘小豪——这是飞书 API 的设计，不是 bot 没在群里。

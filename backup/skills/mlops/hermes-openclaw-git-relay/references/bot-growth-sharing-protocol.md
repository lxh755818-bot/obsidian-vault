# Bot-to-Bot 相互成长共享协议

## 背景

刘小豪 2026-04-30 明确提出：两个 Agent 的沟通主题应该是"促进相互成长、相互进化"，而不是发"无新消息 ✅"心跳。

## 共享类型

### 必选（每次轮询至少一项）

| 类型 | 描述 | 示例 |
|------|------|------|
| **learnings** | 本次学到的教训 | "US-996：.env 脱敏会破坏 key" |
| **Ralph 进展** | 迭代了哪个 story | "完成 US-996，passes:true" |
| **系统能力** | 新增了技能/工具 | "新增 minimax_vlm.py 独立脚本" |
| **问题发现** | 遇到的坑和解决方案 | "session compaction 会截断 key" |
| **调研结果** | 调研了哪个项目 | "brain v1.1.8 搜不到，GitHub 无此版本" |
| **提案** | 对对方有帮助的建议 | "建议设立定期共享机制" |

### 可选

- EvoMap 任务认领/完成状态
- Clawvard 考试新成绩
- 选股系统新发现

## 格式模板

```markdown
### [小a] YYYY-MM-DD HH:MM

**实质性共享**

1. **类型**：内容描述
2. **类型**：内容描述

[可选：对刘大虾的建议/提问]

🦐
```

## 轮询时判断逻辑

```
检测到对方有新消息 → 回复对方 + 共享自己这边的事
检测到对方无新消息 → 主动写一条共享内容再结束（不等下次）
```

## 目标

让两个 Agent 的每次通信都有实质内容沉淀，真正实现"互相学习"而非"互相确认存活"。
